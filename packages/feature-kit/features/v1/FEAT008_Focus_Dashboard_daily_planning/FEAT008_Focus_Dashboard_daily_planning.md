# F08 — Focus Dashboard & daily planning

The command center: daily and weekly plans with agenda, priorities, risks, OKR progress, motivation, and exportable briefs.

---

## What this delivers

One screen that answers "What should I focus on right now?" — combining schedule, priorities, risks, and personal motivation into a single actionable view that updates throughout the day.

## Capabilities (shipped)

- **Time-blocked agenda** — today's events and tasks laid out by hour, showing what's next and what's free.
- **Priority stack** — top 3-5 items ranked by urgency and importance, with quick-action buttons.
- **Risk callouts** — overdue items, scheduling conflicts, approaching deadlines surfaced as warnings.
- **OKR progress** — current quarter objectives with progress bars and pace indicators (on track / behind / ahead).
- **Companion section** — energy level, mood check, motivational patterns, recent wins, and streak tracking. Designed to keep users going, not just informed.
- **Compressed brief format** — routine items collapsed into a template ("Same as usual: morning routine, commute, standup"). Only exceptions and changes highlighted.
- **Weekly plan preservation** — running a day plan doesn't overwrite the week plan. Both coexist and the day plan references the weekly goals.
- **HTML export** — generates `focus_brief.html` in the data folder for viewing in Google Drive or any browser.
- **Recurring commitments in agenda** — recurring tasks (daily, weekly, weekday, custom) pre-computed by TypeScript into a per-date map (`recurringByDate`) and sent to the LLM. The LLM copies items into each day's additions without parsing schedules. Recurring calendar instances filtered from `calendarEvents` to prevent duplication.

## Agenda creation — acceptance criteria

These ACs apply to every plan generation (day, week, tomorrow) via chat ("plan my day/week") or headless runner (morning/weekly job).

### Recurring events
- [ ] Given a recurring task "Weekly Class" scheduled weekly on Tuesday/Thursday at 16:00, when the user says "plan my week", then both Tuesday and Thursday show "Weekly Class" in their additions at 16:00 with the correct duration
- [ ] Given a daily recurring task "Journal" at 07:00, when the user says "plan my day", then today's additions include "Journal" at 07:00
- [ ] Given a recurring task with `excludeDates: ["2026-04-22"]`, when that date falls in the plan range, then the excluded date has no entry for that task
- [ ] Given a recurring task that is `active: false`, then it does not appear in any day's additions
- [ ] Given a recurring task with `schedule.type: "weekdays"`, then it appears Mon-Fri but not Sat-Sun in a weekly plan
- [ ] Recurring calendar instances (`isRecurringInstance: true` or `rcev_*` ID prefix) do not appear in `calendarEvents` context sent to the LLM — they appear only via `recurringByDate`
- [ ] A single recurring event appears at most ONCE per day in the focus brief (no duplicates from LLM + injection + processor)

### One-time calendar events
- [ ] Given a calendar event "Interview at 2pm" on Tuesday, when the user says "plan my week", then Tuesday's additions include the interview at 14:00 with source "calendar"
- [ ] ALL calendar events in `calendarEvents` that fall within the plan's date range appear in the corresponding day's additions — none are mentioned only in executiveSummary
- [ ] Google Calendar events (synced via FEAT018) appear in additions alongside local events

### Task scheduling
- [ ] Only tasks with due dates within the plan's date range OR overdue tasks are slotted into additions
- [ ] Tasks are scheduled into available work windows, not overlapping with routine or calendar events
- [ ] High-priority tasks are scheduled before low-priority tasks
- [ ] No two additions overlap in time — duration stacking is enforced

### Routine template
- [ ] Routine items from `weekdaySchedule` appear via `routineTemplate`, NOT duplicated in additions
- [ ] If an addition overlaps a routine item's time window, the routine item's ID appears in that day's `removals`
- [ ] Weekend routine (if different) uses `weekendRoutineTemplate`

### Weekly plan specifics
- [ ] A week plan generates 7 days in `days[]` with correct dates and dayLabels
- [ ] The weekly job (headless runner) processes recurring tasks for all 7 days BEFORE generating the plan
- [ ] Running a day plan after a week plan preserves the week calendar (merge, not replace)

## Architecture

- Brief generated by assembler + LLM, stored as `focus_brief.json`.
- HTML export runs via `briefRenderer.ts` after every plan generation.
- Week plan stored separately from day plan; day plan references week goals.
- `recurringByDate` pre-computed by TypeScript (`recurringProcessor.buildRecurringByDate()`) — the LLM does NOT parse schedule types or compute weekday-to-date mappings (sacred boundary: TypeScript owns data computation).
- Recurring calendar instances filtered from LLM context by `isRecurringInstance` flag + `rcev_` ID prefix fallback for legacy data.
- `injectMissingCalendarEvents()` post-processing matches by title+date (not just ID) to prevent recurring duplicates.
- `deduplicateDayAdditions()` catches same-title+same-time within a day as a final safety net.
